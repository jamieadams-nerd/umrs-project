= Developer Guide
:description: How to write high-assurance code for UMRS: design patterns, language guides, build tooling, compliance annotations, and the rustdoc API contract.

// NIST 800-218 SSDF PW.1 — Define security requirements for the software
// NSA RTB VNSSA — Non-bypassable security requirements baked into process

This guide is for developers writing code in the UMRS project.
It covers the *why* and *how* of high-assurance development:
what threat each design pattern defeats, how to apply it in Rust, Bash, and Python,
how to annotate code so auditors can trace every security decision to a requirement,
and how to deliver rustdoc API documentation that is itself a compliance artifact.

The API reference (rustdoc) is delivered separately via `cargo doc`.
The doc comments in the source are the authoritative source — this guide
explains how to write them.
