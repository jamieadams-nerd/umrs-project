= umrs-label — Security Label Registry Browser
:description: Overview of the umrs-label tool: interactive browser for UMRS CUI and \
    Canadian Protected label catalogs.
:toc: left
:toclevels: 3

// NIST SP 800-53 AC-3, AC-16, AU-3, SI-10

`umrs-label` displays and browses the UMRS security label registry — the CUI
(Controlled Unclassified Information) catalog and the Canadian Protected catalog.
Operators use it to look up marking details, handling requirements, warning statements,
and MCS category assignments for any label on the system.

The tool has two output modes.
In TUI mode (default when stdout is a terminal), it presents an interactive tree browser
with a detail panel, search, and a security posture header showing hostname, OS,
SELinux mode, and FIPS state.
In CLI mode (`--cli` or when stdout is not a terminal), it renders a human-readable
grouped text listing.

The browser is read-only.
No catalog mutation is possible through the interface.
(NIST SP 800-53 AC-3)

'''

== Configuration and Data Files

`umrs-label` reads JSON reference databases that define the label registry.
No runtime configuration file is required.
Under the FHS 2.3 §4.11 layout, catalog paths default to
`/opt/umrs/share/umrs/`. They can be overridden with `--us-catalog` and
`--ca-catalog`, or collectively redirected with the `UMRS_CONFIG_DIR`
environment variable for development runs.

.Catalog file roles
[cols="2,4", options="header"]
|===
| File | Role

| `US-CUI-LABELS.json`
| Primary US CUI catalog.
  Contains the `markings` object with one entry per CUI marking.
  Each entry carries: `name`, `abbrv_name`, `index_group`, `designation`,
  `level`, `required_warning_statement`, and `required_dissemination_control`.
  Path defaults to `/opt/umrs/share/umrs/US-CUI-LABELS.json`.
  Override with `--us-catalog` or set `UMRS_CONFIG_DIR`.

| `CANADIAN-PROTECTED.json`
| Canadian Protected catalog (Protected A, B, C).
  Uses the same `markings` structure as the US catalog.
  Labels use Treasury Board terminology.
  Path defaults to `/opt/umrs/share/umrs/CANADIAN-PROTECTED.json`.
  Override with `--ca-catalog` or set `UMRS_CONFIG_DIR`.
  Optional: the tool continues if this file is absent.

| `LEVELS.json`
| Shared sensitivity level definitions mapping level identifiers (`s0`–`s3`)
  to human-readable descriptions across all Five Eyes national contexts.

| `US-CUI-PALETTE.json`
| Display color palette for US CUI markings.
  Used by the TUI tree to apply per-marking colors.
  CUI palette colors do not use red or orange, which are reserved for
  classified tiers in Five Eyes systems.

| `/sys/fs/selinux/enforce`
| Live SELinux enforcement mode — read via `SecureReader` (sysfs magic verified).
  Populates the SELinux mode field in the TUI security posture header.

| `/etc/os-release`
| OS identity — read via the TOCTOU-safe `OsDetector` platform abstraction.
  Populates the OS name field in the TUI header.

| `/proc/sys/crypto/fips_enabled`
| FIPS mode state — read via `SecureReader` (procfs magic verified).
  Populates the FIPS state field in the TUI header.

|===

NOTE: All kernel-state reads route through the `umrs-platform` abstraction layer.
Direct `std::fs` reads are not used.
Filesystem magic verification is applied to every `/proc` and `/sys` read.

'''

== Quick Reference

[source,shell]
----
# Browse all labels in TUI mode (default when stdout is a TTY)
umrs-label

# CLI text listing
umrs-label --cli

# Use a custom catalog path
umrs-label --us-catalog /opt/umrs/share/umrs/US-CUI-LABELS.json

# Show help
umrs-label --help
----

.TUI key bindings

== Security Design

*Read-only interface* — the browser cannot create, edit, or delete catalog entries.
The catalog files are opened for reading only.
No write path exists in the interface.
(NIST SP 800-53 AC-3)

*Validated input* — all CLI arguments are validated by clap at entry, before any catalog
I/O or TUI initialization occurs.
Unknown flags cause immediate rejection with a usage error.
(NIST SP 800-53 SI-10)

*Security posture header* — the TUI header shows SELinux mode and FIPS state on every
rendered frame.
Operators can see the platform's security posture without leaving the tool.
(NIST SP 800-53 AU-3)

*CUI palette color policy* — marking display colors do not use red or orange.
In Five Eyes classified environments, red signifies SECRET and orange signifies
TOP SECRET.
Using these colors for unclassified CUI markings would create dangerous visual
confusion for operators who work across classification levels.
