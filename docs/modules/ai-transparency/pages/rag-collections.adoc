= RAG Collections and Their Sources

:toc: left

[id="rag-collections"]

This page is a reference inventory of all RAG collections currently available to UMRS
agents.
For each collection: what it covers, the source documents it was built from, the chunk
count, and which agents use it.

For an explanation of how collections are built and why they exist, see
xref:knowledge-pipeline.adoc[How Knowledge Enters the System].

== The Three-Layer Reference Architecture

The RAG system has three distinct layers, each serving a different purpose:

[cols="1,2,2,2", options="header"]
|===
| Layer | Location | Contents | When to use

| *RAG database*
| Queried via `rag-query` skill
| Chunked, semantically searchable text from all collections
| Default — start here for any technical question

| *Source documents*
| `.claude/references/<collection>/`
| Original PDFs, HTML, Markdown used to build the RAG
| When full document context is needed, or to verify a RAG result's surrounding text

| *Official refs archive*
| `refs/nist/`, `refs/dod/`, `refs/fedramp/`
| Checksummed canonical copies of standards
| Authoritative source of record; integrity-verified via `refs/manifest.md`
|===

Some documents exist in both `refs/` and `.claude/references/`.
The `refs/` copy is authoritative.
The `.claude/references/` copy is the RAG source staging area.

== How to Query

All collections are searchable via the `rag-query` skill:

[source,bash]
----
# Search all collections
python .claude/rag/query.py "NIST SP 800-53 AC-4 information flow enforcement"

# Search a specific collection
python .claude/rag/query.py "MLS range write constraint" --collection selinux-notebook

# List all collections with chunk counts
python .claude/rag/query.py --list-collections
----

See xref:skills-catalog.adoc[Skills Catalog] for full usage documentation.

== Manifest Verification

Every source document in the official refs archive has a SHA-256 checksum in
`refs/manifest.md`.
To verify a document has not been modified since download:

[source,bash]
----
sha256sum refs/nist/sp800-53r5.pdf
# Compare against manifest entry
----

This is the same integrity verification applied to software dependencies.
