= STIG Hardening with OpenSCAP
:description: Applying and verifying the RHEL 10 STIG baseline using OpenSCAP.

After the base RHEL 10 installation, use OpenSCAP to apply the DISA STIG hardening
baseline. This is the preferred approach because it is automated, auditable, and
produces a compliance report suitable for an accreditation package.

== Install OpenSCAP

[source,bash]
----
sudo dnf install -y openscap-scanner scap-security-guide
----

== List Available Profiles

The SCAP Security Guide ships profiles for RHEL 10:

[source,bash]
----
oscap info /usr/share/xml/scap/ssg/content/ssg-rhel10-ds.xml
----

The STIG profile is the preferred baseline for this deployment.

== Evaluate Current Compliance

Run an evaluation to see where the system stands before applying any remediation:

[source,bash]
----
sudo oscap xccdf eval \
    --profile <stig-profile-id> \
    --results /tmp/stig-results.xml \
    --report /tmp/stig-report.html \
    /usr/share/xml/scap/ssg/content/ssg-rhel10-ds.xml
----

Open `/tmp/stig-report.html` in a browser to review findings.

drift. A common approach is to schedule a weekly `oscap eval` and ship the results
to your audit log store.
