= Step 5b: Directory Structure
:toc: left
:description: FHS 3.0 compliant directory layout for UMRS — static files, configuration, variable data, and key material.

UMRS follows the Linux Filesystem Hierarchy Standard 3.0 for add-on packages.
Static files, host-editable configuration, and variable data each live under
separate root directories so that `/opt` can be mounted read-only without
breaking logging or configuration.

Key material is deliberately placed *outside* the add-on package tree. It uses
standard FHS system paths (`/etc/keys/umrs/`, `/var/lib/umrs/keys/`) to satisfy
NIST SP 800-53 CM-6 and SA-8. See
xref:umrs/5b-directory-structure/key-material-trees.adoc[Key Material Trees] for
the full rationale.

== The Three FHS 3.0 Add-on Package Roots

[cols="1,2,2",options="header"]
|===
| Root | FHS Reference | Purpose

| `/opt/umrs/`
| FHS 3.0 §4.5, §5.15
| Static package files: binaries, reference databases, man pages, compiled
  locale catalogs. The package owns this tree. Never modified at runtime.

| `/etc/opt/umrs/`
| FHS 3.0 §3.13
| Host-specific, admin-editable configuration. Policy metadata and tool
  configuration only — no key material.

| `/var/opt/umrs/`
| FHS 3.0 §3.15
| Variable data: logs and state files. Written during normal operation.
|===

From FHS 3.0 §5.15: _"No other package files may exist outside the `/opt`,
`/var/opt`, and `/etc/opt` hierarchies."_ This separation allows the package
tree to be mounted read-only and host-specific configuration to survive a reinstall.
