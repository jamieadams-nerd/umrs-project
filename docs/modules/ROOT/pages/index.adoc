= UMRS Documentation

The Unclassified MLS Reference System (UMRS) is a free, MIT-licensed reference system for
anyone who needs to protect sensitive unclassified information — whether that means following
CMMC, handling U.S. CUI designations, working under Canadian Protected A/B/C requirements,
or simply building software that a security auditor can actually verify. UMRS covers the full
stack: deploying kernel-enforced information labels, running security tools that give operators
evidence-backed answers, and writing high-assurance Rust with patterns that hold up under
scrutiny. The engineering is real. The controls are mapped. The source is open.

== Current Focus — Cantrip Release

The first release, codenamed **Cantrip**, targets RHEL 10 with SELinux in targeted policy
mode. It delivers CUI labeling using the MCS (Multi-Category Security) mechanism, plus two
initial security tools: `umrs-ls` (security-enriched directory listing) and `umrs-state`
(system security posture introspection).

NOTE: Targeted policy mode gives you real MCS labeling and access enforcement now. A future
release will tighten to MLS policy mode for stronger mandatory separation. CUI labeling
on targeted policy is a supported, meaningful step — not a placeholder.

Ubuntu and AppArmor support is planned for the tools and software stack. CUI labeling
remains a RHEL and SELinux capability.

This project uses AI agents throughout — for research, code review, documentation, and
security analysis. The xref:ai-transparency:index.adoc[AI Transparency] section explains
how agents are structured, what each one does, and how their work traces back to
authoritative sources.

== What UMRS Covers

UMRS is four interrelated things you can explore independently or together:

* **Libraries** — reusable Rust crates for SELinux modeling, platform detection, and MLS math
* **Patterns** — high-assurance Rust patterns (TPI, TOCTOU safety, provenance verification) with real codebase examples
* **Tools** — CLI and TUI tools that give security operators typed, evidence-backed answers about what a system is running and whether it meets policy
* **Assessment** — auditor-ready evidence with compliance control mappings (NIST SP 800-53, CMMC, NSA RTB)

This is a reference system — its purpose is to educate, demonstrate, and invite collaboration.
It is not a production hardening tool or a scanner. Pick up what is useful.
