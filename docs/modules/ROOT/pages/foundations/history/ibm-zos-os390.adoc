= IBM Z, z/OS, and Linux on Z

IBM Z is a hardware platform — a mainframe family. It can run multiple, very different operating
systems simultaneously on the same physical machine. Understanding which software layer you are
talking about matters because z/OS and Linux on Z are fundamentally different systems that happen
to share the same hardware.

== IBM Z Platform

IBM Z (formerly called the zSeries) is a mainframe hardware platform. It supports:

* Multiple logical partitions (LPARs) running on the same physical machine
* Both z/OS and Linux workloads, sometimes on the same hardware simultaneously
* Mainframe-specific I/O and virtualization infrastructure (z/VM, KVM on Z)
* Hardware cryptographic accelerators relevant to high-assurance deployment

The hardware itself provides strong isolation guarantees between LPARs — a form of hardware
partition that predates modern hypervisors.

== z/OS

z/OS is IBM's flagship mainframe operating system. It is not Linux, and it is not UNIX in the
traditional sense. It is descended from OS/360 through MVS and OS/390, and it is optimized for:

* Massive batch processing
* Transaction processing via CICS and IMS
* High-reliability enterprise workloads

z/OS uses EBCDIC internally rather than ASCII, operates with JCL (Job Control Language) for
batch job submission, and has a fundamentally different kernel, API surface, and security model
than Linux.

=== z/OS Security Architecture

z/OS implements mandatory-style access controls through the Resource Access Control Facility
(RACF), or through equivalent third-party security managers. Key security characteristics:

* Security labels (SECLABELs) that function similarly to MLS labels
* Compartments and categories that parallel MCS category-based separation
* Trusted execution paths for administrative operations
* Mandatory access controls enforced at the resource manager level

z/OS security is not marketed as "Bell-LaPadula MLS" in the academic sense, but the underlying
concepts — labeled subjects and objects, access decisions based on dominance relationships,
trusted administration paths — are recognizably MLS-adjacent. The emphasis is on
compartmentalization, strong auditing, and trusted administration rather than formal
research-model purity.

In short: z/OS represents decades of MLS-style engineering adapted to enterprise production
realities. The concepts survived, even as the terminology evolved.

== Linux on IBM Z

Linux on IBM Z is standard Linux — not a fork or an IBM-specific variant. It is:

* The standard Linux kernel, compiled for the s390x architecture
* Standard enterprise distributions (RHEL, SLES, Ubuntu) repackaged for s390x
* Architecturally identical to RHEL or Ubuntu on x86_64 or aarch64 — same kernel,
  same userspace tools, same SELinux support

IBM provides hardware enablement, crypto accelerators, I/O drivers, and virtualization
layers. The distribution itself — Red Hat, SUSE, or Canonical — remains the standard product.

|===
| Distribution | Architecture | Notes

| Red Hat Enterprise Linux
| s390x
| Full support, SELinux, FIPS mode, DNF/YUM — common in regulated environments

| SUSE Linux Enterprise Server
| s390x
| Long-standing IBM partnership; strong presence in banking and finance; SELinux and AppArmor

| Ubuntu Server
| s390x
| Supported by Canonical; common for cloud-native and development workloads

| Debian, Fedora
| s390x
| Community-supported; not typically used in production on IBM Z
|===

Rocky Linux and AlmaLinux are not commonly supported on IBM Z at this time. The ecosystem
focuses on RHEL, SLES, and Ubuntu for regulated environments.

== Relationship to UMRS Concepts

For UMRS purposes, IBM Z matters as an architectural reference point in two ways.

First, z/OS demonstrates that MLS-style thinking can survive at scale in production environments
spanning decades. RACF with SECLABELs, compartmentalization, and trusted administrative paths
are the same ideas that UMRS applies to RHEL 10 — adapted to a different operating system and
a different era.

Second, Linux on IBM Z is a supported target for RHEL-compatible deployments. UMRS components
that run on RHEL on x86_64 run on RHEL on s390x with the same SELinux policy, the same MCS
labeling mechanism, and the same high-assurance tooling. The s390x hardware provides additional
cryptographic accelerators and hardware isolation that strengthen the platform posture.

The correct framing for any UMRS deployment on IBM Z is: "Linux on IBM Z runs standard RHEL
built for the s390x architecture." The underlying UMRS mechanisms are identical.
