== About This Library

These patterns are the design vocabulary of the UMRS codebase. They are not optional
style guidelines — they are the mechanisms by which the codebase satisfies its NSA RTB,
NIST 800-53, and SSDF obligations at the implementation level.

Each page covers:

. *The threat* — what goes wrong when the pattern is absent
. *The pattern* — exactly what the code does and why
. *In the UMRS codebase* — real source examples with module and file references
. *When to apply* — the conditions that trigger this pattern
. *Controls* — which NIST 800-53 and NSA RTB controls the pattern satisfies
. *Summary* — a one-paragraph statement suitable for a design review or blog post

The patterns are organized into four groups below. The implementation status column
is maintained so the xref:devel:index.adoc[rust-developer] can identify patterns
that are documented but not yet applied.

For the full consolidated narrative with inline code examples for all patterns in
one document, see xref:devel:high-assurance-patterns.adoc[High-Assurance Coding Patterns].

== Open — Awaiting Implementation

The following patterns are documented, required, and not yet applied in the codebase.
The rust-developer should look for opportunities to implement them — particularly before
the `vaultmgr` crate handles real key material or credentials.

=== Zeroize Sensitive Data

*Where it is needed:* `vaultmgr` — any type that holds cryptographic key material,
session tokens, or cleartext credentials.

*What is missing:* The `zeroize` crate is not in the workspace. No type in `vaultmgr`
implements `Zeroize` or `ZeroizeOnDrop`.

*Before shipping:*

. Add `zeroize = "1"` to `vaultmgr/Cargo.toml` after running `cargo-audit`
. Derive `#[derive(ZeroizeOnDrop)]` on every type that holds key material
. Explicitly call `.zeroize()` on local stack buffers holding decrypted content before
  they go out of scope

See xref:pattern-supply-chain.adoc[Supply Chain Hygiene] for the dependency evaluation
checklist before adding `zeroize`.

=== Constant-Time Comparison

*Where it is needed:* Any code path that compares a user-supplied token, MAC, HMAC,
or password hash against a stored value.

*What is missing:* The `subtle` crate is not in the workspace. No comparison in the
codebase uses `ConstantTimeEq`.

*Before shipping:*

. Add `subtle = "2"` to the relevant crate's `Cargo.toml` after running `cargo-audit`
. Replace any `==` comparison on credential byte sequences with `.ct_eq().into()`

See xref:pattern-supply-chain.adoc[Supply Chain Hygiene] for the dependency evaluation
checklist before adding `subtle`.
