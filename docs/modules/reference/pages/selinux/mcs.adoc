= Multi-Category Security (MCS) Reference
:description: How MCS works in SELinux and how mcstransd translates category labels.

Multi-Category Security (MCS) is an SELinux access control mechanism that provides
non-hierarchical isolation between processes and objects using category labels.

== How MCS Works

* A process can only access an object if it has a *superset* of the object's categories.
  A file labeled `s0:c1,c2` denies access to a process with only `c1`.
* Categories range from `c0` to `c1023`.
* Unlike MLS, MCS uses a single sensitivity level (`s0`) — there is no hierarchical
  "Top Secret vs. Unclassified" ranking, only horizontal compartmentalization.

MCS is widely used in containerization (Podman, Docker) and virtualization (sVirt),
where each container gets a unique random category pair (e.g., `s0:c12,c45`) to
prevent cross-container access even after an escape.

== The Translation Daemon: mcstransd

SELinux stores categories internally as raw numbers (`s0:c0,c1`). `mcstransd` translates
these into human-readable labels for display in tools like `ls -Z` and `ps -Z`.

=== Configuration

Mappings are defined in `setrans.conf`:

[source]
----
/etc/selinux/<POLICYTYPE>/setrans.conf
----

Example mapping:

[source]
----
s0:c0=Marketing
s0:c0,c1=Engineering
----

After defining these, `ls -Z` shows `Marketing` instead of `c0`.

=== Communication

`mcstransd` communicates with `libselinux` via a Unix domain socket at:

[source]
----
/var/run/setrans/.setrans-unix
----

=== Useful Commands

[source,bash]
----
# List all current translations
chcat -L

# Check daemon status
systemctl status mcstrans
----

== UMRS Usage

UMRS uses MCS category labels to represent CUI compartments. The `setrans.conf`
shipped with UMRS maps categories to CUI label names. See
xref:deployment:rhel/rhel10-setrans.adoc[CUI Label Configuration] for installation steps.
