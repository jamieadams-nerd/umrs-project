== Intent

Multi-Level Security has its roots in classified information handling.
Many MLS implementations and most MLS documentation assume a classified
environment: SECRET and TOP SECRET levels, SCI compartments, and cleared
users operating under strict need-to-know rules enforced by a security
officer.

UMRS targets a different baseline: *Controlled Unclassified Information*
(CUI). CUI is not classified. It does not carry hierarchical classification
levels in the traditional sense. It uses category-based handling
requirements governed by NIST SP 800-171 and CMMC 2.0 — not the
Intelligence Community's classification authorities.

This distinction matters architecturally. Designs that assume a classified
environment carry assumptions that do not transfer cleanly to CUI:

* Classification levels (UNCLASSIFIED / SECRET / TOP SECRET) map
  awkwardly onto CUI categories (PRVCY, ITAR, CTI, etc.)
* SCI compartment semantics differ from CUI subcategory semantics
* Cleared-user models differ from CUI authorized-holder models
* Classification authority chains differ from CUI policy frameworks
