= Directory Purpose Matrix
:toc: left
:description: Single reference table -- every UMRS path with owner, mode, SELinux type, and purpose.

This table is the single authoritative reference for the on-disk layout.
Every path declared in `umrs.fc.in` and every directory created by the
installer appears here.

FHS 3.0 references: §3.5 (etc), §3.13 (etc/opt), §3.15 (var), §4.5 (opt),
§4.11 (share), §5.15 (opt packages).

IMPORTANT: As of 2026-04-16 the fcontext entries for `/etc/opt/umrs/` and
`/var/opt/umrs/` are present in `umrs.fc.in`. The pre-FHS paths
(`/opt/umrs/etc`, `/opt/umrs/var`) have been removed from the template.
Run `semodule -i umrs.pp` after rebuilding to activate the updated entries,
then run `restorecon -RFv /opt/umrs /etc/opt/umrs /var/opt/umrs`.

5+h| UMRS Key Material Root (`/etc/keys/umrs/`) — NIST SC-12, SP 800-57 §5.3

| UMRS key material namespace root. No listing by non-root.

| `/etc/keys/umrs/sealing/`
| `root:root` | `0700` | `umrs_seal_key_t`

| `/etc/keys/umrs/signing/`
| `root:root` | `0700` | `umrs_sign_key_t`

| `/etc/keys/umrs/wrapping/`
| `root:root` | `0700` | `umrs_kek_t`

| `/etc/keys/umrs/staging/`
| `root:root` | `0700` | `umrs_sign_key_t`
| Pre-activation key material (SP 800-57 §5.3 pre-active state).
  NIST SC-12, CM-3.

5+h| UMRS Runtime Key State (`/var/lib/umrs/keys/`) — NIST SC-12, AC-3 — _(not yet created by installer)_

| `root:root` | `0700` | `umrs_retired_key_t`

== Compliance Mapping

[cols="2,3",options="header"]
|===
| Control | Coverage

| NIST SP 800-53 CM-2 (Baseline Configuration)
| Installer captures the authoritative static-file baseline.

| NIST SP 800-53 CM-6 (Configuration Settings)
| Layout is fixed by FHS 2.3; no site variance permitted.

| NIST SP 800-53 CM-7 (Least Functionality)
| Only classified artifact types are staged or installed.

| NIST SP 800-53 CM-8 (Component Inventory)
| `EXPECTED_BINARIES` in the stager declares the full artifact set.

| NIST SP 800-53 SA-12 (Supply Chain Protection)
| Staging is the IMA-signing checkpoint; no stray files reach signing.

| NIST SP 800-53 AC-3, SC-12, SC-28
| Key material at rest under `/etc/keys/umrs/` and `/var/lib/umrs/keys/`.
  Distinct SELinux types per key function.

| NIST SP 800-53 MP-6, MP-7
| Retired key staging under `/var/lib/umrs/keys/retired/` with destruction service requirement.

| NIST SP 800-57 Pt 1 Rev 5 §5.3
| Directory layout reflects SP 800-57 key lifecycle states (pre-active, active,
  suspended, deactivated/retired).

| FHS 3.0 §3.5, §3.13, §3.15, §4.5, §4.11, §5.15
| On-disk layout conforms. Deliberate deviations from §4.5 add-on package autonomy
  are documented in xref:umrs/5b-directory-structure/key-material-trees.adoc[Key Material Trees §6].
|===
