= Signing and Verifying Audit Log Chains
:toc: left
:description: A prototype Bash script for signing rotated audit logs, maintaining a chain index, and verifying individual logs and the full chain.

This page documents a prototype Bash signing and verification script for auditd log
files. The script maintains a chain database linking each signed log to the previous
entry, enabling detection of file tampering, deletion, or reordering.

== Assumptions

* Audit logs: `/var/log/audit/audit.log`, `audit.log.1`, `audit.log.2`, ...
* Private key: `/etc/pki/ha-sign/auditd/private/auditd_signing_key`
* Public key: `/etc/pki/ha-sign/auditd/public/auditd_signing_key.pub`
* Chain index: `/var/lib/ha-sign/auditd-chain/chain.db`
* Signature files stored next to the logs with `.sig` appended.

== Chain Database Format

The chain database is a plain-text file with pipe-delimited records:

[source]
----
<seq>|<log_path>|<log_sha256>|<sig_path>|<prev_log_sha256>
----

The `prev_log_sha256` field for the first record is `NONE`.

== Subcommands

=== sign

Signs a rotated audit log and appends a chain record.

[source,bash]
----
ta_audit_sign.sh sign /var/log/audit/audit.log.1
----

Steps performed:

1. Verifies the log file exists and the keys are readable.
2. Computes `SHA-256` of the log file.
3. Creates a detached signature using `openssl dgst`.
4. Reads the last sequence number and chain hash from the database.
5. Appends a new chain record linking this log to the previous one.

Expected output:

[source]
----
Signed /var/log/audit/audit.log.1
  Signature: /var/log/audit/audit.log.1.sig
  Hash:      <sha256-hex>
  Chain seq: 7, prev_hash=<previous-sha256>
----

=== verify-file

Verifies the signature on a single log file and checks its hash against the chain record.

[source,bash]
----
ta_audit_sign.sh verify-file /var/log/audit/audit.log.1
----

Steps performed:

1. Verifies the log file and signature file exist.
2. Calls `openssl dgst -verify` against the public key.
3. If the file appears in the chain database, recomputes the hash and compares it to
   the stored chain record.

Expected output on success:

[source]
----
OK: Signature verification succeeded for /var/log/audit/audit.log.1
OK: Hash matches chain record for /var/log/audit/audit.log.1 (seq=7)
----

=== verify-chain

Verifies the entire chain from start to finish.

[source,bash]
----
ta_audit_sign.sh verify-chain
----

For each record in the chain database, the script:

1. Checks that the log file and signature file exist.
2. Verifies the signature.
3. Recomputes the hash and checks it against the chain record.
4. Verifies that the `prev_log_sha256` field matches the hash from the previous record.

Expected output on success:

[source]
----
Verifying full chain from /var/lib/ha-sign/auditd-chain/chain.db ...
Checking seq=1 log=/var/log/audit/audit.log.5
  OK
Checking seq=2 log=/var/log/audit/audit.log.4
  OK
...
Full chain verification: OK
----

== Script Configuration Block

The following configuration variables are set at the top of the script:

[source,bash]
----
AUDIT_LOG_DIR="/var/log/audit"
KEY_ROOT="/etc/pki/ha-sign"
AUDIT_KEY_NAME="auditd_signing_key"
PRIVKEY="${KEY_ROOT}/auditd/private/${AUDIT_KEY_NAME}"
PUBKEY="${KEY_ROOT}/auditd/public/${AUDIT_KEY_NAME}.pub"
CHAIN_DIR="/var/lib/ha-sign/auditd-chain"
CHAIN_DB="${CHAIN_DIR}/chain.db"
DIGEST_ALG="sha256"
----

Adjust these paths to match your HA-Sign layout.

== Security Requirements

The script must run as root (or effective root) to access private keys. The chain
database file is created at `0600` permissions. Signature files are created at `0600`
permissions.

When integrating with `logrotate`, call this script from a `postrotate` block:

[source]
----
postrotate
    /usr/sbin/service auditd reload >/dev/null 2>&1 || true
    /usr/local/sbin/ta_audit_sign.sh sign /var/log/audit/audit.log.1
endscript
----

== Compliance Alignment

[cols="1,3"]
|===
| Control | Satisfaction

| `AU-9` (Protection of Audit Information)
| Detached signatures prove logs have not been modified since signing.

| `AU-10` (Non-repudiation)
| Signatures are tied to the audit signing key; key ownership is documented by HA-Sign.

| `AU-11` (Audit Record Retention)
| Chain verification confirms the full retention sequence is intact with no gaps.

| `SI-7` (Software and Information Integrity)
| Chaining detects deletion or reordering that individual signatures alone cannot.
|===
