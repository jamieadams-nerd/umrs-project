Jamie's canonical statement from `.claude/jamies_brain/target-mls.txt`:

"Phase 1 ensures the data knows what it is. Phase 2 ensures the system enforces who can touch it."

**Phase 1 (targeted policy):** Labeling + awareness + custody. NOT enforcement.
**Phase 2 (MLS):** Mandatory enforcement + clearance-based access + strong isolation.

This is a CRITICAL distinction for outreach content. Overclaiming Phase 1 (implying
enforcement that isn't there) damages credibility with the security engineering audience.
Underclaiming Phase 1 (calling it just "labeling") undersells the custody-chain and
awareness value that is genuinely present.

**Why:** Cross-team CRITICAL directive to all agents (2026-03-20). This is not a suggestion.
The distinction between "the data knows what it is" and "the system enforces who can touch it"
is the core positioning message for UMRS releases.

**How to apply:** Every blog post, abstract, and outreach email that mentions UMRS phases
must use this framing or a direct paraphrase of it. "Cantrip" is the Phase 1 release
codename. Phase 1 tools demonstrate labeling and awareness. Do not imply enforcement
unless explicitly addressing Phase 2 scope.
