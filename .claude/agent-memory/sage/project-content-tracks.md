## Track 2: AI Journey Series
- Jamie's first project using AI agents — periodic reflections on what worked
- Topics include: feedback loops, swim buddy concept, guardrails for agent errors/hangs, agent specialization, the human-in-the-loop model
- Audience: developers curious about AI-assisted development, open source practitioners
- Tone: honest, personal, technique-focused — not hype, not a product pitch

**How to apply:** When drafting an editorial calendar, allocate slots for both tracks. AI journey posts can fill gaps between heavier technical posts. They also serve as a soft introduction to the project for readers who arrive via the AI angle. Do not conflate the two tracks — each has its own audience and purpose.

Source material for AI journey posts: agent aliases, swim buddy rule, feedback loop patterns, permission tuning, RAG workflow — all documented in cross-team notes and Jamie's brain files.
