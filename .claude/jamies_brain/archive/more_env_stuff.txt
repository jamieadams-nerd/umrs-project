1. Formal Standards (canonical baseline)

These give you the only truly “standardized” env vars.

POSIX / Single UNIX Specification
	•	The Open Group
	•	Spec: Base Specifications Issue 7 / POSIX.1-2017

Where to look:
	•	“Environment Variables” section
	•	Individual utility specs (many define vars implicitly)

Coverage:
	•	PATH, HOME, IFS, LANG, LC_*, TZ, etc.

Value:
	•	Ground truth for standard = POSIX
	•	Small set, but high assurance

⸻

2. GNU / glibc + dynamic linker (critical real-world surface)

glibc

Where to extract:
	•	glibc manual → “Environment Variables”
	•	ld.so / dynamic linker docs

Key goldmine:
	•	man ld.so
	•	glibc source: elf/rtld.c, sysdeps/...

Coverage:
	•	LD_PRELOAD
	•	LD_LIBRARY_PATH
	•	LD_AUDIT
	•	LD_DEBUG
	•	MALLOC_*

Value:
	•	Security-critical variables
	•	Must-have for UMRS classification

⸻

3. Linux man-pages project (aggregated truth)

Linux man-pages project

This is one of your BEST ingestion targets.

Where:
	•	man 7 environ
	•	Individual man pages (many define env vars)

Strategy:
	•	Parse all man* pages for:

ENVIRONMENT
ENVIRONMENT VARIABLES



Coverage:
	•	Cross-cutting: libc, tools, kernel interfaces

Value:
	•	Broad, semi-authoritative, machine-extractable

Coverage:
