## Cryptography Acceleration Extensions

### AES-NI
- Hardware acceleration for AES encryption and decryption.
- Provides instructions that execute AES rounds directly in hardware.
- Improves encryption throughput dramatically and reduces side-channel risk compared to software implementations.
- Example instructions:
  - AESENC
  - AESENCLAST
  - AESDEC
  - AESDECLAST
  - AESKEYGENASSIST
- AES instruction sets accelerate encryption rounds and key generation operations.  [oai_citation:0‡Wikipedia](https://en.wikipedia.org/wiki/AES_instruction_set?utm_source=chatgpt.com)

---

### VAES (Vector AES)
- Vectorized AES instructions.
- Found in AVX-512 implementations.
- Allows multiple AES blocks to be processed simultaneously.

---

### SHA Extensions
- Hardware acceleration for SHA-1 and SHA-256 hashing.
- Used by cryptographic libraries and TLS stacks.

---

### CLMUL / PCLMULQDQ
- Carry-less multiplication instruction.
- Used for:
  - Galois Field math
  - AES-GCM authentication

---

## Hardware Randomness

### RDRAND
- Hardware random number generator.
- Returns random values generated by an on-chip entropy source.

---

### RDSEED
- Returns raw entropy suitable for seeding cryptographic PRNGs.
- Intended for stronger entropy initialization.

---

## Vector Processing Extensions

These extensions significantly accelerate cryptographic libraries.

### SSE family

Includes:

- SSE
- SSE2
- SSE3
- SSSE3
- SSE4.1
- SSE4.2

Used heavily in:

- hashing
- block cipher pipelines
- crypto library vectorization

---

### AVX
- Advanced Vector Extensions.
- Introduces 256-bit vector registers.

---

### AVX2
- Expanded integer vector operations.
- Used by modern cryptographic libraries.

---

### AVX-512
- 512-bit vector operations.
- Introduces mask registers and expanded SIMD functionality.  [oai_citation:1‡Wikipedia](https://en.wikipedia.org/wiki/AVX-512?utm_source=chatgpt.com)
- Used in high-performance cryptography and parallel data processing.

---

## Big Integer / Multi-Precision Extensions

### ADX
- Multi-precision arithmetic instructions.
- Improves big integer operations used in:
  - RSA
  - ECC

---

### BMI1 / BMI2
- Bit manipulation instructions.
- Improve performance of:
  - crypto primitives
  - hashing algorithms

---

## Trusted Execution and Confidential Computing

### Intel SGX
- Software Guard Extensions.
- Allows execution inside secure enclaves protected from the OS and hypervisor.
- Provides memory encryption and attestation capabilities.  [oai_citation:2‡Intel](https://www.intel.com/content/www/us/en/products/docs/accelerator-engines/software-guard-extensions.html?utm_source=chatgpt.com)

---

### AMD SEV
- Secure Encrypted Virtualization.
- Encrypts virtual machine memory with per-VM keys.
- Prevents hypervisors from reading guest memory.  [oai_citation:3‡AMD](https://www.amd.com/en/developer/sev.html?utm_source=chatgpt.com)

---

### SEV-ES
- Encrypted CPU register state.

---

### SEV-SNP
- Adds stronger VM integrity protections.

---

### Intel TDX
- Trust Domain Extensions.
- Hardware isolation for virtual machines.

---

## Memory Encryption Extensions

### AMD SME
- Secure Memory Encryption.
- Transparent memory encryption using a hardware key.

---

## Security Key Management

### Intel Key Locker
- Hardware protected key storage.
- Prevents software access to raw encryption keys.
