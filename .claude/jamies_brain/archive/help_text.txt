Concise / plain-language

Each filesystem in Linux has a unique “magic number” — a built-in identifier the kernel uses to recognize its type. By checking this value on a mount point, we can confirm that /proc is actually the proc filesystem and not something else pretending to be it.

Slightly more technical

Linux assigns a fixed “magic number” to every filesystem type (e.g., procfs, sysfs, ext4). When we validate /proc, we read this identifier from the kernel and verify it matches the expected procfs value, ensuring the mount point is genuine and not substituted or tampered with.
