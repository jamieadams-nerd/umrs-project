High-Assurance Linux Capability Matrix

Purpose

The capability matrix enumerates Linux kernel and system security features that significantly affect the assurance properties of a system, including integrity, enforcement non-bypassability, tamper resistance, and provenance.

These capabilities differ from traditional compliance hardening rules in that they represent architectural security mechanisms rather than configuration baselines.

The matrix allows a security auditor to determine:
	•	what assurance mechanisms exist
	•	whether they are active
	•	how they affect trust boundaries
	•	whether they can be bypassed

⸻

Capability Classification Model

Each capability should be classified along several axes.

Field	Purpose
capability_id	Stable identifier
subsystem	Kernel / boot / SELinux / logging
assurance_property	Integrity / isolation / provenance
runtime_probe	Where to read state
desired_state	Expected value
bypassability	Can attackers disable it
persistence_method	How it survives reboot
security_impact	Explanation

Major Assurance Capability Domains

1 Kernel Enforcement Controls

These are kernel mechanisms that restrict privileged behavior.

Capability	Probe Location	Desired State
Kernel Lockdown Mode	/sys/kernel/security/lockdown	integrity or confidentiality
Disable Module Loading	/proc/sys/kernel/modules_disabled	1
Kexec Disabled	/proc/sys/kernel/kexec_load_disabled	1
BPF JIT Harden	/proc/sys/net/core/bpf_jit_harden	2
Unprivileged BPF Disabled	/proc/sys/kernel/unprivileged_bpf_disabled	1

Assurance property:
	•	prevents runtime kernel modification
	•	reduces attack surface for root compromise

2 Boot Chain Integrity

Ensures system trust begins at boot.

Capability	Probe	Desired

Kernel Lockdown Auto Mode	kernel command line	enforced

Assurance property:
	•	prevents unsigned kernel execution
	•	protects kernel memory integrity

Capability Metadata Example

A structured catalog entry might look like this:

{
  "capability_id": "KERNEL_MODULES_DISABLED",
  "subsystem": "kernel",
  "assurance_property": "integrity",
  "probe": "/proc/sys/kernel/modules_disabled",
  "desired_state": "1",
  "bypassability": "non-bypassable once enabled",
  "persistence": "kernel runtime irreversible",
  "impact": "prevents runtime kernel extension loading"
}

This makes the matrix machine queryable.
