  Elena — "I was doing it by instinct and taste, without a theoretical foundation"

  Biggest surprise: The problems she deals with daily — synonymy ("security label" vs "security
  context"), homographs ("category" meaning different things to SELinux and general readers),
  navigation structure — are the same problems Ranganathan was solving in 1937. "The problem is
  ancient. Our instance of it is not special. That was humbling."

  Sharpest insight: The NISO controlled vocabulary standard reframed terminology maintenance from a
  style decision to a retrieval contract. If an auditor searches for "sensitivity level" and half
  the pages say "classification level," we've broken a promise.

  What challenged her: She assumed Diataxis was theoretically grounded. Turns out Svenonius had a
  more rigorous version decades earlier. Diataxis is a useful packaging, not an endpoint.

  Her top suggestions:
  1. Glossary scope notes — every term needs notes that say what it covers AND what it doesn't.
  "Security context" needs a note explaining how it differs from "security label."
  2. Audience-aware entry points — the navigation gives all three audiences (engineers, auditors,
  adopters) the same scent. It should say "if you are an auditor, start here." This is a
  domain-analysis problem she'd been treating as a layout problem.

  ---
  Rusty — "I had never thought about a command-line security tool as a genre before"

  Biggest surprise: Miller's genre theory hit him unexpectedly. umrs-uname isn't defined by its form
   (TUI with tabs); it's defined by the recurring social situation it responds to: the morning
  security check. "What does this situation demand? Fast orientation, not comprehensive coverage."

  Sharpest insight: The Gulf of Evaluation from MIT 6.831 maps directly onto what UMRS does. The
  kernel has state the operator can't see. umrs-uname is literally a "Gulf-narrowing device."
  Surfacing contradictions isn't a feature — it's the core obligation.

  What challenged him: Shannon entropy applied to indicator display. On a well-configured system,
  most indicators are green (low entropy). A green indicator carries almost no information. A red
  one carries a lot. "We should be ranking by information content, not by category." The anomaly
  should interrupt the operator, not wait to be found by scrolling.

  His top suggestions:
  1. Pinned summary redesign — the summary pane should answer "is this machine safe right now?"
  without scrolling. Currently some critical signals require scrolling to verify. The affordance
  question is: what does the operator come here to do?
  2. Group headings as purpose, not location — "Kernel Attributes (/sys)" is a location. "Memory
  Integrity" is a purpose. The group name is the chunking mechanism — a bad name literally costs
  cognitive capacity.

  Herb — "The giddiness is real. The concerns are also real. Both are true simultaneously."

  Biggest surprise: The SSP is not a compliance checklist — it's a story. The FedRAMP playbook
  literally says "it is telling a story about the security of your CSO." If any thread breaks, the
  reviewer can't trace the argument and the package stalls.

  Sharpest insight: His code annotation audits have been checking for citation presence when they
  should be checking for citation quality. A bare // NIST SP 800-53 SI-7 is exactly the bad control
  implementation statement the FedRAMP playbook warns against — restating the requirement without
  describing how it's met.

  What challenged him: The assessment engine's 5-layer model maps cleanly to FedRAMP artifacts (he
  built a mapping table), but it's missing a common_controls field. umrs-platform provides common
  security infrastructure that all tools inherit — the accreditation framework has explicit
  machinery for this (SP 800-18 §2.5.3) and the assessment engine should use it.

  His top suggestions:
  1. Annotation quality standard — shift from "does a citation exist?" to "does the annotation
  describe who, what, where, when, why and how the control is met?" The tire pressure analogy from
  the FedRAMP playbook is his new benchmark.
  2. common_controls in the assessment engine — the BundleManifest has profile_used but no field for
   controls inherited from the platform. An AO reviewing the package needs to see this separation
  explicitly.
