NAME
    prctl - operations on a process or thread

LIBRARY
    Standard C library (libc, -lc)

SYNOPSIS
    #include <linux/prctl.h>  /* Definition of PR_* constants */
    #include <sys/prctl.h>

    int prctl(int op, ...);

DESCRIPTION
    The prctl() function manipulates various aspects of the behavior of the
    calling thread or process. It is invoked with a first argument describing
    the operation to perform, followed by additional arguments whose
    significance depends on that first argument.

SECCOMP-RELEVANT OPERATIONS

PR_SET_SECCOMP
    Set the seccomp mode of the calling process. The second argument must
    be one of:

    SECCOMP_MODE_STRICT (1)
        Only the following system calls are permitted: read(2), write(2),
        _exit(2), sigreturn(2). Any other call terminates the process with
        SIGKILL. Equivalent to seccomp(SECCOMP_SET_MODE_STRICT, 0, NULL).

    SECCOMP_MODE_FILTER (2)
        Install a BPF filter passed via the third argument (a pointer to
        struct sock_fprog). Requires CAP_SYS_ADMIN or no_new_privs=1.
        Equivalent to seccomp(SECCOMP_SET_MODE_FILTER, 0, filter).

PR_GET_SECCOMP
    Return the seccomp mode of the calling process. The result is:
        0  if seccomp is not enabled
        1  if SECCOMP_MODE_STRICT
        2  if SECCOMP_MODE_FILTER

PR_SET_NO_NEW_PRIVS
    Set the no_new_privs attribute for the calling thread. With this bit
    set, the process and its children cannot gain additional privileges
    via execve(2) (e.g., no setuid or file capabilities take effect).

    This is a prerequisite for installing seccomp filters without
    CAP_SYS_ADMIN. Once set, cannot be unset.

    prctl(PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0);

PR_GET_NO_NEW_PRIVS
    Return 1 if no_new_privs is set; 0 otherwise.

RETURN VALUE
    On success, the function returns a nonnegative value. On error, it
    returns -1 and sets errno to indicate the error condition.

ERRORS
    EINVAL: The value of op is not recognized or not supported on this system.
    EINVAL: An unused argument is nonzero.

VERSIONS
    Linux 2.1.57. glibc support from 2.0.6.

STANDARDS
    Linux (non-standard)

CAVEATS
    The libc wrapper uses a variadic argument list, requiring arguments to be
    passed with correct width. Numeric constants should include a suffix
    (e.g., "0L"). Careless use of certain prctl() operations can disrupt the
    user-space runtime environment, warranting careful application.
