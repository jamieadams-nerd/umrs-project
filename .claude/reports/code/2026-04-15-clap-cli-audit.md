## Per-tool findings

### umrs-c2pa — LOW

- clap derive struct present with rich subcommands (`config {validate,generate}`, `creds {generate,validate}`) and global flags (`--config`, `--verbose`).
- `--verbose` is global — matches the Universal Verbose rule.
- Default config path: `umrs-c2pa.toml` in CWD. **Violates `/opt/umrs` default-path rule** (memory: `project_config_default_location`). Should fall back to `/opt/umrs/etc/umrs/umrs-c2pa.toml`.
- Flag `chain_json` → clap generates `--chain-json` (kebab-case) — correctly documented in man page.
- Guest-coder's 2026-04-02 report referenced a broken `--detailed-json` flag. **Not present in current source.** Either fixed silently or misidentified. **Not a live bug.**
- Antora page uses `--config` and flags correctly.

**Actions:**
1. Change `--config` default from `umrs-c2pa.toml` → `/opt/umrs/etc/umrs/umrs-c2pa.toml` with CWD fallback documented.
2. Confirm in task log that `--detailed-json` finding is resolved.

### umrs-label — MED

- clap derive. Flags: `--us-catalog`, `--ca-catalog`, `--cli`, `--json`.
- **No `--verbose` flag** — violates Universal Verbose rule.
- Default catalog paths: `config/us/US-CUI-LABELS.json`, `config/ca/CANADIAN-PROTECTED.json` — **violates /opt/umrs default-path rule**.
- Blocks i18n (task #8): zero gettext wrapping in source (separate issue, tracked).

**Actions:**
1. Add `--verbose` / `-v` using umrs-c2pa's `verbose.rs` pattern.
2. Change defaults to `/opt/umrs/etc/umrs/us/US-CUI-LABELS.json` and `/opt/umrs/etc/umrs/ca/CANADIAN-PROTECTED.json`.

### umrs-stat — MED

- clap derive. Positional `PATH`, flag `--json`.
- **No `--verbose` flag.**
- setrans discovery path for mcstransd: needs verification that it honors `UMRS_CONFIG_DIR` / `/opt/umrs`. Likely reads from kernel-gated path — out of scope for this audit.

**Actions:**
1. Add `--verbose` / `-v`.

### umrs-ls — HIGH

- **Hand-rolled `std::env::args()` parsing.** No clap. Knox C-10 already filed a TODO on this 2026-04-09.
- Flags parsed: `--json`, `--cli`, `--flat`, `--no-iov`, `--no-mtime`, `--with-size`, `--with-inode`, `--color`.
- **No `--help`** — operator running `umrs-ls --help` will see nothing (or the path `--help` interpreted as target directory).
- **No `--version`.**
- **No `--verbose`.**
- Man page and Antora page document the flags correctly, but operators have no in-tool discovery.

**Actions (single migration pass):**
1. Migrate to clap derive (derive `Parser`). Preserves all current flags verbatim.
2. Add `--help`, `--version` (both auto-generated by clap).
3. Add `--verbose` / `-v` using umrs-c2pa's `verbose.rs` pattern.
4. Honor `/opt/umrs/etc/umrs/` for catalog lookups in the detail popup (memory: `project_umrs_ls_display_reuse` + `project_config_default_location`).
5. Add CLI smoke tests in `tests/cli_args_tests.rs` matching umrs-label's pattern (2026-04-13 precedent).

### umrs-uname — HIGH

- **No argument parsing at all.** `main()` takes no arguments from the command line.
- No `--help`, no `--version`, no `--verbose`, no `--json`, no `--cli`.
- Man page SYNOPSIS correctly shows `umrs-uname` with no options — accurately reflects current state — but this IS the gap.

**Actions:**
1. Add clap derive with `--help`, `--version`, `--json`, `--cli`, `--verbose`, `-v`.
2. If `--json` is deferred to a later phase, declare it and emit "reserved for future implementation" at runtime (same pattern as umrs-label/umrs-stat).
3. Update man page OPTIONS section once flags land.
4. Update Antora Quick Reference once flags land.
