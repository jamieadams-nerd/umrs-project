# The Diátaxis Compass and Map

Source: https://diataxis.fr/compass/ and https://diataxis.fr/map/
Retrieved: 2026-03-12

## The Compass

The Diátaxis compass functions as a decision-making tool for documentation. As the page explains, "A map is most powerful in unfamiliar territory when we also have a compass to guide us."

The compass reduces documentation decisions to two fundamental dimensions, helping authors determine what type of documentation is needed or how to improve existing content.

### The Four Documentation Types (Truth Table)

| Content Focus | User Need | Documentation Type |
|---|---|---|
| Informs action | Skill acquisition | Tutorial |
| Informs action | Skill application | How-to guide |
| Informs cognition | Skill application | Reference |
| Informs cognition | Skill acquisition | Explanation |

### Using the Compass

The decision process requires answering two core questions:

1. **Action or cognition?** (practical steps vs. theoretical knowledge)
2. **Acquisition or application?** (learning vs. working)

Flexible terminology:
- *Action*: "practical steps, doing"
- *Cognition*: "theoretical or propositional knowledge, thinking"
- *Acquisition*: "study"
- *Application*: "work"

The compass works best when doubt or difficulty arises. Users can apply it at multiple scales—examining individual sentences or entire documents—and from different angles.

## The Map

Diátaxis describes a **two-dimensional structure** for organizing documentation rather than a simple list. The framework positions different documentation types in relationship to each other, creating a mental map.

### The Problem of Structure

Documentation frequently fails due to poor structural organization. Without clear architecture, creators often default to organizing content around product features—an approach that rarely succeeds.

A structured approach provides two critical advantages:
- **Clear expectations** for readers regarding content purpose and placement
- **Clear guidance** for authors on how content should be written

### Documentation Type Comparison

| Aspect | Tutorials | How-to Guides | Reference | Explanation |
|--------|-----------|---------------|-----------|-------------|
| **Answers** | "Can you teach me to…?" | "How do I…?" | "What is…?" | "Why…?" |
| **Purpose** | Provide learning experience | Help achieve specific goals | Describe machinery | Illuminate topics |
| **Form** | Lesson | Series of steps | Dry description | Discursive explanation |

### Blur

Documentation types naturally blur into adjacent categories. Common collapses include:
- Tutorials and how-to guides merge into undifferentiated content
- Reference and explanation blend together

"When these distinctions are allowed to blur, the different kinds of documentation bleed into each other."

### The Journey Around the Map

Expert development follows a meaningful cycle:

1. **Learning-oriented phase**: Begin with hands-on instruction
2. **Goal-oriented phase**: Apply skills to specific objectives
3. **Information-oriented phase**: Consult technical reference when knowledge gaps appear
4. **Explanation-oriented phase**: Reflect on practice and understanding conceptually
