Retrieved: 2026-03-12

## Diátaxis: A Systematic Approach to Technical Documentation

Diátaxis is a methodology for technical documentation authoring that addresses content, style, and architecture through understanding user needs.

The framework derives its name from Ancient Greek: _dia_ ("across") and _taxis_ ("arrangement").

## Core Concept

The approach identifies four distinct documentation needs and corresponding formats:

- **Tutorials**: Learning-oriented content
- **How-to guides**: Task-focused instructions
- **Technical reference**: Information lookup resources
- **Explanation**: Conceptual understanding

These four forms are organized in a systematic relationship where documentation should be structured around the patterns of user requirements.

## Key Benefits

For documentation users, Diátaxis addresses:
- What content to include
- How to write it effectively
- How to organize it logically

For creators and maintainers, the framework is:
- Lightweight and easy to understand
- Straightforward to implement
- Implementation-agnostic
- Provides quality guidance for evaluating documentation

## Proven Application

The framework has been successfully adopted across hundreds of documentation projects. Organizations including Vonage, Gatsby, and Cloudflare have reported that Diátaxis improved documentation clarity, user experience, and contributor engagement.
