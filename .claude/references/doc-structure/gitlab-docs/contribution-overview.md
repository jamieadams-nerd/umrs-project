# Contributing to GitLab Documentation — Overview

Source: https://docs.gitlab.com/development/documentation/
Retrieved: 2026-03-12

## Overview

The GitLab documentation serves as the authoritative reference for configuring, using, and troubleshooting GitLab. Community contributions are welcomed.

## Update the Documentation

**Prerequisites:**
- Complete contributor onboarding at contributors.gitlab.com to access community forks

**Steps to contribute:**

1. Navigate to the `/doc` directory in the GitLab community fork
2. Locate your target page (check docs.gitlab.com URL path or select "View page source" at page bottom)
3. Select **Edit** > **Edit single file**
4. Make your changes
5. Enter a commit message (3-5 words, capitalized, no period)
6. Select **Commit changes**
7. Apply the **Documentation** template to your merge request
8. Write a brief summary and link related issues
9. Select **Create merge request**

## What to Work On

Documentation updates don't require an issue, but curated items for new contributors are available.

The Vale linting tool can help identify and fix documentation issues.

## Get Help

Consult the Technical Writing team for:
- Placement guidance
- Outline discussion
- General assistance

Identify your stage group's Technical Writer via the GitLab handbook, then assign or mention them in issues/merge requests.
