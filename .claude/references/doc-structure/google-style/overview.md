# Google Developer Documentation Style Guide — Overview

Source: https://developers.google.com/style
Retrieved: 2026-03-12

## Overview

This style guide provides editorial standards for creating clear, consistent technical documentation aimed at software developers and technical practitioners.

## Getting Started

**Key entry points for new users:**
- Highlights
- Voice and tone
- Text-formatting summary

For specific questions, use this guide as a reference document (e.g., consult the word list for terminology).

## Editorial Resources

### Reference Hierarchy

Follow this priority order when seeking guidance:

1. **Project-specific style** - Follow your project's unique requirements first
2. **This style guide** - Apply these guidelines if project guidance is unavailable
3. **Third-party references** - Use these as final resources:
   - *Spelling:* Merriam-Webster.com
   - *General style:* The Chicago Manual of Style (17th edition)
   - *Technical style:* Microsoft Writing Style Guide

Additional resources include Apple's style guide and Red Hat's supplementary style guide.

## Philosophy: Breaking the Rules

> "Break any of these rules sooner than say anything outright barbarous." —George Orwell

This guide offers recommendations, not rigid rules. Prioritize clarity and consistency for your specific audience and domain, even when departing from these guidelines. Maintain internal consistency when deviating from recommendations.
