# Write the Docs Software Documentation Guide — Overview

Source: https://www.writethedocs.org/guide/
Retrieved: 2026-03-12

## Overview

This is a living, breathing guide gathering collective wisdom about creating software documentation and technical writing. The guide expanded from helping developers understand documentation to serving a broader community.

## How to Start Writing Technical Documentation

- **How to write software documentation**: Covers why documentation matters, what to include, next steps, and tools for getting started
- **Getting started best practices**: Guidance for open-source developers and developers justifying documentation at companies
- **Building documentation mindshare**: Strategies including problem definition, engineering engagement, input gathering, taxonomy building, and regular meetings
- **Documentation principles**: General principles, content quality guidelines, source considerations, publication standards, and body content best practices
- **Conquering imposter syndrome**: Addresses self-doubt through perspectives on accessibility, knowledge gaps, and documentation as product

## Resources for Creating Documentation

- **Style guides**: Traditional resources, sample guides, accessibility, bias considerations, API documentation, and command-line resources
- **Accessibility guidelines**: Introductions, web accessibility resources, and relevant talks
- **Reducing bias**: Guidance on inclusive example names
- **Support team documentation**: Tips for cross-functional work between support and technical writing
- **UX writing**: Videos, blogs, podcasts, books, and online courses

## Approaches to Creating Documentation

- **Docs as Code**: Conference talks and articles on treating documentation like software
- **DocOps**: Overview of documentation operations practices and resources

## Understanding Markup Languages

- **Markdown**: Introduction, benefits, and usage
- **reStructuredText**: Features and resources
- **XML**: Applications and resources
- **AsciiDoc**: Capabilities and references

## Documentation Tools

- **Tool selection**: Questions and considerations for choosing platforms
- **Writing tools**: Sphinx and alternatives; testing and quality resources
- **SEO**: Optimization strategies for documentation discoverability

## API Documentation

- **Tools and platforms**: Apiary, API Blueprint, and testing approaches
