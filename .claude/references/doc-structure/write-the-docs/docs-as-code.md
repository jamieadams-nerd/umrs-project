# Docs as Code — Write the Docs

Source: https://www.writethedocs.org/guide/docs-as-code/
Retrieved: 2026-03-12

## Overview

Documentation as Code (Docs as Code) represents a philosophy for creating documentation using the same tools and workflows as software development. This approach integrates writers with development teams and fosters shared ownership of documentation quality.

## Core Tools and Practices

The Docs as Code approach utilizes:

- Issue tracking systems
- Version control via Git
- Plain text markup languages including Markdown, reStructuredText, and Asciidoc
- Code review processes
- Automated testing

## Key Benefits

Implementing this methodology provides several advantages:

- Writers achieve better integration within development teams
- Developers frequently contribute initial documentation drafts
- Documentation requirements can be enforced during code merge processes, encouraging developers to document features while knowledge is fresh

## Recommended Resources

Three books offer comprehensive guidance on implementation:

1. *Docs Like Code: Collaborate and Automate to Improve Technical Documentation* by Anne Gentle
2. *Modern Technical Writing: An Introduction to Software Documentation* by Andrew Etter
3. *Crafting Docs for Success: An End-to-End Approach to Developer Documentation* by Diana Lakatos

Additionally, [docToolchain](https://doctoolchain.github.io/docToolchain/) provides an open-source toolchain demonstrating practical implementation.

## Write the Docs Conference Presentations

The community has featured numerous talks on this topic from 2015-2022, including presentations from organizations like Google, Rackspace, Microsoft, AWS, and the UK Government Digital Service, documenting their adoption journeys and lessons learned.
