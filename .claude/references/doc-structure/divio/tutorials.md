# Tutorials — Divio Documentation System

## Overview

Tutorials serve as hands-on lessons guiding beginners through practical projects. They are fundamentally **learning-oriented**, focusing on *how* to accomplish tasks rather than *what* concepts mean.

> "You are the teacher, and you are responsible for what the student will do."

The primary purpose is enabling learners to understand remaining documentation and software after completing the tutorial. Poor or absent tutorials significantly hinder user acquisition.

## Key Principles for Writing Effective Tutorials

### Learning Through Doing

> "In the beginning, we only learn anything by doing"

Tutorials must engage learners in practical activities, progressing from simple to complex operations. This mirrors how humans naturally acquire skills.

### Getting Started

Beginning steps can be basic and hand-held. The tutorial need not demonstrate expert or optimal practices—its goal is starting the learner's journey, not reaching perfection.

### Ensuring Functionality

> "The single most important thing is that what you ask the beginner to do must work."

Learners must see expected results from their actions. Without live instruction, tutorials must prevent errors through careful design.

### Immediate Visible Results

> "Everything the learner does should accomplish something comprehensible, however small."

Each action should produce visible, understandable outcomes with clear connections to the steps taken.

### Repeatability

Tutorials must work reliably across different operating systems, experience levels, and tools. Regular testing is essential for maintenance.

### Concrete Over Abstract

> "Tutorials need to be concrete, built around specific, particular actions and outcomes."

Learning progresses from concrete examples to abstraction. Premature abstract concepts obstruct understanding.

### Minimal Explanation

> "Don't explain anything the learner doesn't need to know in order to complete the tutorial."

Only essential information belongs in tutorials; detailed explanations should link to other documentation sections.

### Task Focus

Tutorials should concentrate on necessary steps only, ignoring alternative methods or advanced options that distract from the core objective.
