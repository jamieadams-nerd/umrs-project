# The Documentation System — Divio

## Overview

The Divio Documentation System presents a foundational theory about effective software documentation. Rather than treating documentation as a single entity, this framework identifies four distinct types, each serving different purposes and requiring different approaches.

## The Four Types of Documentation

The system organizes documentation into four categories:

1. **Tutorials** - Learning-focused content
2. **How-to Guides** - Task-oriented instructions
3. **Technical Reference** - Detailed information lookup
4. **Explanation** - Conceptual understanding

## Core Principle

As noted in the framework, "there isn't one thing called *documentation*, there are four." Each category has unique objectives and creation methods that must be understood to produce quality documentation.

## Key Benefits

Implementing this system offers significant advantages:

- Improves documentation quality substantially
- Enhances project and product success
- Provides a universally applicable scheme
- Has been successfully adopted across diverse fields and applications

## Underlying Philosophy

The system is based on simple but rarely articulated principles about how documentation functions. These principles remain largely unspoken yet consistently prove effective when applied in practice.

The framework has achieved widespread adoption among both large enterprises and small projects, across open-source and proprietary contexts, demonstrating its practical value and flexibility.
