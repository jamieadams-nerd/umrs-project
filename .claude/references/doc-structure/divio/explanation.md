# Explanation Documentation — Divio Documentation System

## Overview

Explanatory documentation **clarifies and illuminates particular topics** while broadening coverage of subjects. These sections prioritize understanding over task completion.

## Key Characteristics

Explanations are discursive in nature, allowing documentation to step back from software implementation and examine topics from broader perspectives. They're designed for leisurely reading rather than quick reference during coding work.

Topics aren't defined by specific tasks or learning objectives, but rather by "what **you** think is a reasonable area to try to cover at one time."

## Purpose and Value

Like culinary writing examining food through historical, scientific, and technological lenses, explanations analyze subjects from multiple angles. They deepen knowledge and understanding—even when that knowledge isn't immediately applicable—by exploring design decisions, historical context, and technical constraints.

## Writing Guidelines

**Provide Context**: Explanations serve as ideal spaces for background information and contextual details, including the reasoning behind design decisions.

**Discuss Alternatives**: These sections can explore multiple approaches to the same question and evaluate competing opinions or methodologies.

**Avoid Instruction**: Explanations should not teach procedures or provide technical reference material, as other documentation sections handle those functions.

## Integration

Explanatory content often appears scattered throughout documentation or under alternative headings like "Background" or "Key Topics." Effective explanation sections provide the broader picture that enriches user understanding beyond immediate practical needs.
