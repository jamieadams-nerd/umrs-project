### Numbers and Symbols {#letter-number data-text="Numbers and Symbols" tabindex="-1"}

\+ [[link]{.material-icons aria-hidden="true" translate="no"}](#+)
:   OK to use *+* with numbers in text, such as *customer records with 300+ demographic attributes*, except in formal contexts.

[]{.icon-avoid}& (ampersand) [[link]{.material-icons aria-hidden="true" translate="no"}](#ampersand)
:   Don\'t use *&* instead of *and* in headings, text, navigation, or tables of contents.
:   It\'s OK to use *&* when referencing UI elements that use *&*, or in table headings and diagram labels where space constraints require abbreviation.
:   It\'s OK to use `&`{translate="no" dir="ltr"} for technical purposes in code.

2-Step Verification [[link]{.material-icons aria-hidden="true" translate="no"}](#2-step-verification)
:   When referring to Google\'s [2-Step Verification](https://www.google.com/landing/2step/), use initial caps.
:   When referring to [generic 2-step verification](http://searchsecurity.techtarget.com/definition/two-step-verification), use lowercase.

### A {#letter-a data-text="A" tabindex="-1"}

a and an [[link]{.material-icons aria-hidden="true" translate="no"}](#a-an)
:   Use *a* when the next word starts with a consonant *sound*, regardless of what letter it starts with. For more information, see [Articles (a, an, the)](/style/articles).

A/B testing [[link]{.material-icons aria-hidden="true" translate="no"}](#ab)
:   Capitalize and use slash notation for *A/B*.

[]{.icon-avoid}abnormal [[link]{.material-icons aria-hidden="true" translate="no"}](#abnormal)
:   Don\'t use to refer to a person.
:   OK to use to refer to a condition of a computer system.

[]{.icon-avoid}abort [[link]{.material-icons aria-hidden="true" translate="no"}](#abort)
:   Avoid in general usage. Instead, use words like *stop*, *exit*, *cancel*, or *end*. In Linux, *abort* refers to a type of signal that terminates an abnormal process.

about versus on [[link]{.material-icons aria-hidden="true" translate="no"}](#about-on)
:   When a cross-reference includes information that describes what the cross-reference links to, use *about* instead of *on*.
:   [Recommended]{.compare-better}: For more information about indexes, see [Managing indexes](https://cloud.google.com/firestore/docs/query-data/indexing).
:   [Not recommended]{.compare-worse}: For more information on indexes, see [Managing indexes](https://cloud.google.com/firestore/docs/query-data/indexing).

[]{.icon-avoid}above [[link]{.material-icons aria-hidden="true" translate="no"}](#above)
:   Don\'t use for a range of version numbers. Instead, use [*later*](#later).
:   Don\'t use to refer to a position in a document. Instead, use *earlier* or *preceding*.
:   Don\'t use to refer to a position in the UI. Instead, write instructions that avoid directional language. For more information, see [Writing accessible documentation](/style/accessibility).
:   It\'s OK to use *above* in a non-directional way, such as when describing a hierarchy.

[]{.icon-avoid}access (verb) [[link]{.material-icons aria-hidden="true" translate="no"}](#access)
:   Avoid when you can. Instead, use friendlier words like *see*, *edit*, *find*, *use*, or *view*.

access token [[link]{.material-icons aria-hidden="true" translate="no"}](#access-token)
:   Lowercase except at the beginning of a sentence, heading, or list item.

If you aren\'t sure what noun *this* or *that* refers to, then consider rephrasing---otherwise, your reader probably won\'t know what noun you\'re referring to, either.
