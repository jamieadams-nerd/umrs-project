## Items that are sometimes in code font {#items-that-are-sometimes-in-code-font data-text="Items that are sometimes in code font" tabindex="-1"}

The following list includes items that are sometimes in code font, but it\'s not an exhaustive list.

-   **Boolean values**. If you refer directly to a Boolean data type value (such as `true`{translate="no" dir="ltr"} or `false`{translate="no" dir="ltr"}, or `1`{translate="no" dir="ltr"} or `0`{translate="no" dir="ltr"}), then format the value as code. If you refer to the evaluation of a Boolean condition as true or false, then refer to the evaluation in non-code font.

    ::: example
    [Recommended]{.compare-better}:

    -   If the update succeeds, returns `true`{translate="no" dir="ltr"}.
    -   `enableCertificateValidation`{translate="no" dir="ltr"}: If true, validates the SSL certificate before proceeding. If false, trusts the certificate without validating it.
    :::

-   **Command-line utility names**. Often, command-line utility names are spelled the same as the software project or product with which they are associated, with only differences in capitalization. In such cases, use code font for the command and ordinary font for the name of the project or product.

    ::: example
    [Recommended]{.compare-better}:

    -   Invoke the GCC 8.3 compiler using `gcc`{translate="no" dir="ltr"} for C programs or `g++`{translate="no" dir="ltr"} for C++ programs.
    -   To send the file over FTP with IPv6, use `ftp -6`{translate="no" dir="ltr"}.
    -   The options for the `curl`{translate="no" dir="ltr"} command are explained on the curl project website.
    -   The `apt`{translate="no" dir="ltr"} program includes commands from the `apt-get`{translate="no" dir="ltr"} and `apt-cache`{translate="no" dir="ltr"} programs for working with APT packages.
    :::

-   **Email addresses as input or output**. If you want the reader to use the email address as computer input or output, use code font. If you want the reader to treat the email address as a way to contact someone or a reference to someone, use non-code font and hyperlink the email address.

    ::: example
    [Recommended]{.compare-better}:

    -   Enter the username, not the full email address. For example, enter `alex`{translate="no" dir="ltr"}, not `alex@example.com`{translate="no" dir="ltr"}.
    -   For help, contact <support@example.com>.
    :::

## HTTP status codes {#http-status-codes data-text="HTTP status codes" tabindex="-1"}

To refer to a single status code, use the following formatting and phrasing:

an HTTP `400 Bad Request`{translate="no" dir="ltr"} status code

In particular, call it a *status code* instead of a *response code* or *error code*, and put the number and the name in code font. If the *HTTP* is implicit from context, you can leave it out.

To refer to a range of codes, use the following form:

an HTTP `2xx`{translate="no" dir="ltr"} or `400`{translate="no" dir="ltr"} status code

In particular, use *`N`{.variable translate="no"}xx* (with a specific digit in place of `N`{.variable translate="no"}) to indicate *anything in the `N`{.variable translate="no"}00-`N`{.variable translate="no"}99 range*, and put the status code number in code font even if you\'re leaving out the code\'s name.

If you prefer to specify an exact range, you can do so:

an HTTP status code in the `200`{translate="no" dir="ltr"}-`299`{translate="no" dir="ltr"} range

Here, too, put the numbers in code font.
:::

## []{.icon-android}Linking API terms in Android {#linking-api-terms-in-android data-text="Linking API terms in Android" tabindex="-1"}


-   service
-   fragment
-   view
-   loader
-   action bar
-   intent
